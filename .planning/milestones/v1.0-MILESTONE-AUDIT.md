## Tech Debt

### Phase 3: Post-Processing Consistency and Cleanup (2 items, non-blocking)

1. **Cell 10 mu/sigma shadowing on re-execution** — Cell 10 (pre-training EDA) assigns `mu = np.mean(log_delta_np)` and `sigma = np.std(log_delta_np)` as numpy float64 values. In linear top-to-bottom execution this is harmless (Cell 15 overrides with torch tensors). If a user re-executes Cell 10 after Cell 15 during interactive exploration, `full_denorm_pipeline` call sites receive numpy scalars instead of torch tensors. (Tangentially affects QUAL-03, BUG-03 — linear execution is correct.)

2. **Cell 37 diagnostic noise range mismatch** — The diversity diagnostic cell uses `np.random.uniform(0, 2 * np.pi, ...)` while all training/eval/standalone paths correctly use `[0, 4*pi]` per QC-04. Cell 37 generates 5 test outputs for diversity checking only — it does not affect training or evaluation results. (Cosmetically affects QC-04 — all functional paths correct.)

**Total: 2 items across 1 phase**
