  <action>
Four localized fixes in cell 23 (qGAN class) and cell 17:

**BUG-04 — Loss values as Python floats:**
Find all places where loss values are appended to history lists (`self.critic_loss_avg.append(...)`, `self.generator_loss_avg.append(...)`, or similar). Ensure each uses `.item()` to convert from tensor to Python float. Some may already use `.item()` — verify and fix any that don't. This prevents the computation graph from being retained in memory, avoiding OOM over long training runs.

Check lines approximately at: loss storage after critic training step, loss storage after generator training step, and any loss storage in the evaluation/metrics block.

**BUG-05 — Epoch condition:**
Find the line `if epoch % 100 == 0 or epoch + 1 == 3000:` (approximately line 1350 in cell 23). Change `3000` to `self.num_epochs`. This ensures the condition works correctly regardless of the configured number of epochs.

**BUG-06 — Scope delta as self.delta:**
1. In cell 23, add `delta` as a constructor parameter to `qGAN.__init__()` with a default value of 1: `def __init__(self, ..., delta=1):`
2. Store as `self.delta = delta` in __init__
3. In `_train_one_epoch()`, replace all bare references to `delta` (the global variable from cell 17) with `self.delta`. The main usage is in the `lambert_w_transform(generated_data, delta)` call — change to `lambert_w_transform(generated_data, self.delta)`
4. In cell 24 (qGAN instantiation), add `delta=1` to the constructor call (or whatever value is currently in cell 17's global)
5. Cell 17's `delta = 1` global can remain (other code may reference it) but the qGAN class must use self.delta internally

**PERF-02 — Wrap evaluation in torch.no_grad():**
In `_train_one_epoch()`, find the evaluation/metrics computation block (approximately lines 390-435 in cell 23). This block runs AFTER the optimizer steps, generating samples and computing metrics (EMD, stylized facts). Wrap this entire block in `with torch.no_grad():`.

The `torch.no_grad()` context should start AFTER the last optimizer.step() call and end BEFORE the method returns or the next epoch begins. Do NOT wrap the training steps (critic forward/backward/step or generator forward/backward/step) — only the evaluation/inference code.

Also check post-training cells (cells 29, 32-35) for forward passes that should be wrapped in torch.no_grad(). Any cell that calls the generator for inference/visualization (not training) should use torch.no_grad().

**Do NOT change:** Training hyperparameters, loss functions, model architecture, or any value that affects training dynamics.
