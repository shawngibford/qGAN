<objective>
Switch the QNode from parameter-shift to backprop and convert all four per-sample Python loops to single batched QNode calls. This is the full implementation of REG-02 and REG-03 -- no training run needed here, that comes in Plan 02.

Purpose: PennyLane issue #4462 documents that parameter-shift explicitly disallows gradients through broadcasted tapes (PR #4480). Backprop is the only viable diff_method for batched execution with correct gradient flow. The broadcasting conversion replaces O(batch_size) sequential QNode calls with a single call using PennyLane's native parameter broadcasting (shape `(num_qubits, batch_size)` triggers internal vectorized simulation), targeting ~12x speedup.

Output: `qgan_pennylane.ipynb` with backprop QNode, shots=None device, and all four loops converted to single batched calls.
</objective>
