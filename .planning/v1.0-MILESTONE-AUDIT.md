### Integration Issues

**MISS-01: mu/sigma variable shadowing (WARNING)**

Cell 15 correctly unpacks `norm_log_delta, mu, sigma = normalize(log_delta)`. However:
- Cell 16 overwrites: `mu = torch.mean(norm_log_delta).item()` (≈ 0.0), `sigma = torch.std(norm_log_delta).item()` (≈ 1.0)
- Cell 18 overwrites again: `mu = torch.mean(transformed_norm_log_delta).item()`, `sigma = torch.std(transformed_norm_log_delta).item()`

By the time training starts (Cell 33), the module-level `mu`/`sigma` hold Lambert-W-transformed statistics, not the original normalization constants. Both `full_denorm_pipeline` calls (training eval and standalone gen) receive the same wrong values, so BUG-03 consistency is maintained, but the absolute denormalization scale is incorrect.

**Fix:** Rename visualization variables in Cells 16/18 to `mu_viz`/`sigma_viz`, or protect the normalization constants with distinct names like `norm_mu`/`norm_sigma`.
